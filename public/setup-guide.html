2) Run Locally

Use any static server (don‚Äôt open index.html by double-click ‚Äî that breaks routing/assets). 
# Option A: Node http-server
npx http-server -p 8080

# Option B: serve
npx serve -l 8080

# Option C: Python
python3 -m http.server 8080
Then visit http://localhost:8080.

3) Firebase Setup

3.1 Create project

  1. Create a Firebase project.
  2. Add a Web App and copy the config snippet (apiKey, authDomain, etc.).
  3. Paste it into app.js (firebaseConfig).

3.2 Enable Auth

  1. Authentication ‚Üí Sign-in method ‚Üí enable Email/Password.

3.3 Realtime Database

  1. Create Realtime Database (pick a region).
  2. App writes under tenants/<uid>/kv/<key>.

Cloud Sync is optional ‚Äî the app works fully offline with LocalStorage.

3.4 Optional: Firebase Hosting

npm i -g firebase-tools
firebase login
firebase init hosting     # project, "public" = .
firebase deploy

5) App Features

SPA Router: Home, Dashboard, Inventory, Products, COGS, Tasks, Settings, static pages.
Auth: Firebase + Local fallback; session persistence.
Inventory & Products: CRUD, CSV export, images (Data URLs), hover enlarge, product card modal.
COGS: Daily rows with totals and gross profit.
Tasks: Kanban with drag & drop and mobile tap-to-move.
Posts: Simple newsroom with images.
Search: Global search across all entities.
Contact: EmailJS (optional) with mailto fallback.

6) Hot Music Videos

Weekly rotation from a library in app.js, with Shuffle and auto-skip of blocked videos (blacklist).

7) Images & Thumbnails

Uploads are Data URLs stored with the items; hover previews float above tables; mobile opens a modal.

8) Service Worker / PWA

GET-only cache with SW; register via a safe GET; installable via manifest.webmanifest.

9) Contact

  1. Create EmailJS service/template.
  2. Put PUBLIC_KEY, SERVICE_ID, TEMPLATE_ID in app.js.

  • If images don‚Äôt preview, check CSS z-index and overflow.
  • If tasks don‚Äôt drag, ensure dragover uses preventDefault().
