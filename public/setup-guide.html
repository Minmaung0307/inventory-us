1) Files & Structure

Put these files at your site root:

  • index.html
  • styles.css
  • app.js
  • service-worker.js
  • manifest.webmanifest
  • /icons/ (PNG icons + favicon.svg)
  • (Optional) policy.html, license.html, guide.html ‚Äî or keep your own docs.

2) Running locally

Serve with any static server (PWA needs HTTP/HTTPS, not file://):

# Python 3
python -m http.server 9005

# or Node
npx http-server -p 9005

Open http://localhost:9005/.

3) Firebase (optional, for Cloud Sync)

  1. Create a project at Firebase Console.
  2. Enable Email/Password Sign-in in Authentication.
  3. Enable Realtime Database (test rules for development).
  4. Copy the web config into app.js (the firebaseConfig object).
  5. Sign in (top-right ‚Üí Sign In). Turn Cloud Sync ON in Settings to sync local data to your Firebase DB.

4) PWA install button + omnibox icon

  • We provide manifest.webmanifest + a service-worker.js with a fetch handler.
  • Chrome/Edge will show the native ‚ÄúInstall app‚Äù icon in the address bar when the app meets criteria.
  • We also render a custom Install button in the topbar when installation is available.

5) Assets & Images

Uploads are stored locally as DataURLs. If you want shared images across devices, host images in Firebase Storage or on your own CDN and paste the URL into the image field.

6) Features Overview

  • Inventory & Products with images (hover preview + phone modal)
  • COGS with month-to-date snapshots on Dashboard
  • Tasks board with drag-and-drop
  • Search across posts, items, products, users, and pages
  • PWA: offline-friendly, installable
  • Hot Music (YouTube) on Home ‚Äî weekly rotation and auto-skip broken videos
